```
## 规范驱动开发 (SDD)
本项目采用规范驱动开发流程。

- **SDD 流程文档**: docs/sdd/README.md
- **模块状态总览**: docs/sdd/STATUS.md

- **模块规范目录**: docs/sdd/specs/

新功能开发流程: 先写规范 → 实现 → 测试 → 验证。

## 进度追踪（自动维护）

完成任何模块或功能的实现后，Claude Code **必须** 自动更新以下文件，无需用户提醒：

### 必须更新的文件

1. **`TODO.md`** — 将对应条目从 `[ ]` 改为 `[x]`
2. **`docs/sdd/STATUS.md`** — 更新模块状态矩阵（状态、实现路径、测试路径），若是新模块则从「待规范模块」移入矩阵
3. **`.claude/projects/.../memory/MEMORY.md`** — 更新「最近完成」和「下一步优先级」

### 触发条件

以下任何事件发生时，**立即**执行上述更新：

| 事件 | TODO.md | STATUS.md | MEMORY.md |
|------|---------|-----------|-----------|
| 完成一个模块/功能的实现 | `[x]` 打勾 | 状态 → Impl/Verified | 更新「最近完成」 |
| 编写或更新了模块规范 | — | 状态 → Draft/Review | 更新「最近完成」 |
| 修复了 bug | 相关条目打勾或新增 `[x]` | 备注栏补充 | 更新「最近完成」 |
| 添加了测试 | — | 测试路径列更新 | — |
| 重构了代码 | — | 实现路径列更新（如有变化） | 记录重构内容 |
| 新增了依赖或做了架构决策 | — | — | 更新「关键约定」 |
| 阶段性里程碑完成 | 整个 Phase 标记完成 | 批量更新 | 更新「下一步优先级」 |

### 更新规则

- **即时性**: 完成动作后立即更新，不要积攒到最后
- **准确性**: 只更新实际完成的内容，不要提前标记
- **一致性**: 三个文件的信息必须互相一致，不能出现 TODO 已打勾但 STATUS 未更新的情况
- **原子性**: 每次更新三个文件视为一个整体操作，要么全部更新，要么都不更新

这是强制流程，不可跳过。
```
