# Architecture Decision Records (ADR)

## What is an ADR

An Architecture Decision Record (ADR) documents important architectural decisions made in a project. Each ADR describes the context of a decision, the options considered, the final decision, and its consequences.

## Why ADRs

- Record the **reasons** behind decisions, not just the outcomes
- New team members (or your future self) can quickly understand why a particular approach was chosen
- Avoid re-debating decisions that have already been made
- Provide architectural context to Claude Code for more consistent code generation

## Status Definitions

| Status | Meaning |
|--------|---------|
| **Proposed** | Proposed, pending discussion |
| **Accepted** | Accepted, being implemented |
| **Deprecated** | Deprecated, superseded by a new ADR |
| **Superseded** | Superseded by a specified new ADR |
